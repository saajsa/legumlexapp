
Introduction
The Customers Perfex API operates over HTTPS and uses JSON as its data format. The API is a RESTful API and utilizes HTTP methods and HTTP status codes to specify requests and responses. A token is bound to a whole Perfex installation. To get started using the API you first need an API token.

Installation
As long as you download our API module from CodeCanyon, extract its contens.

You will notice a folder called "documentation" and a new zip file, called "upload.zip".

Since "documentation" folder contains files that are not needed in your Perfex CRM's installation, we will focus on the "upload.zip" file.

File upload.zip contains the module files (in a module format) that you upload in Perfex CRM's Modules installation section.

API activation
Step 1) Go to your Perfex CRM's Admin area and select the following menu item: SETUP â†’ MODULES. Step 2) Select the extracted upload.zip at Module installation selection prompt and press INSTALL. Step 3) Find the newly installed module in the list, press ACTIVATE and enter your license key. Step 4) Optionally enable "Allow Users to Register" in the settings of the module (Customers REST API menu)

Obtaining a token
Your customers can obtain an API token that will be used in your commands, by triggering the Login endpoint. Login endpoint always returns "token" parameter in its response, that can be used in all Customer API Commands. If your customers are not registered in Perfex CRM's area, they will need to use Register endpoint prior Logging in.

Note that in order to use the registration endpoint, an admin needs to enable "Allow Customers to register using API") in the settings of the module.
Usage of the API
Available commands of the API are described below, along with their request-response examples.

We strongly suggest that you hire a professional, in order to have your third party connection done, if you are not familiar with APIs, as we cannot guide you through the basic understanding of REST APIs (that is a requirement in order to use this item and falls completely outside the scope of our support plan terms of it).

End of documentation intro part. Below, you will find all available commands that can be used with our API, along with their methods.
